# PC题词器产品需求文档（PRD）

**产品名称**：Teleprompter Pro  
**版本**：V2.0 专业版  
**文档版本**：1.0  
**最后更新**：2025年11月  
**定位**：专业级演讲和短视频口播题词工具

---

## 第一部分：产品概述

### 1.1 产品定义

Teleprompter Pro 是一款为**演讲者和内容创作者**设计的专业题词工具。通过智能的界面设计和人性化的功能，帮助用户在会议发言、短视频口播、直播等场景中保持自然表现，提升表现力。

### 1.2 核心价值主张

| 对象 | 核心需求 | 我们的解决方案 |
|------|---------|-------------|
| **会议演讲者** | 需要自然发言，不想一直看稿 | 智能换行 + 视线引导，保持眼神接触 |
| **短视频创作者** | 需要多次重复拍摄，控制节奏 | 表现力评分 + 节奏提示，快速成片 |
| **直播主播** | 需要实时控制内容，不出错 | 快速导航 + 预读显示，信心十足 |
| **销售团队** | 需要统一演讲稿，快速上手 | 脚本库 + 团队管理，高效协作 |

### 1.3 用户画像

- **主用户**：知识工作者、销售、讲师、创作者（25-45岁）
- **次用户**：企业培训部、市场部团队负责人
- **使用频度**：周1-3次（高频场景），月2-10次（一般场景）
- **支付意愿**：为提升专业形象付费，年均预算¥200-500

## 第二部分：产品形态设计

### 2.1 整体架构

```
应用主界面
├── 菜单栏（顶部）
├── 主工作区
│   ├── 左侧边栏（脚本库/快捷操作）
│   ├── 中央显示区（题词显示）
│   └── 右侧工具栏（场景模式/设置）
└── 底部控制条（播放控制/时间显示）
```

### 2.2 窗口形态定义

#### **形态一：标准工作窗口**（默认，宽屏推荐）

**尺寸规范**：1600×1000px（最小1280×720px）

**布局特性**：
- **左侧边栏**：280px 宽度（脚本库 + 快捷工具）
- **中央显示区**：960px 宽度（题词内容显示）
- **右侧工具栏**：360px 宽度（模式选择 + 实时参数）

**视觉特点**：
- 深色主题（类似VS Code，护眼且专业）
- 清晰的层级感，减少视觉干扰
- 核心区域（中央显示）占视觉权重60%

**典型应用场景**：
- 台式电脑用户
- 需要全功能操作的专业用户
- 需要监控表现力数据的用户

---

#### **形态二：极简显示窗口**（全屏演讲模式）

**尺寸规范**：全屏或指定分辨率（1920×1080 或 2560×1440）

**布局特性**：
- **隐藏所有工具栏**，仅显示题词内容
- **黑色背景** + 大字号白文本（对比度最大化）
- **底部控制条**：仅显示进度条、计时器、快捷键提示（40px 高度）

```

│      （字号48-72px，居中对齐）      │
```

**交互特点**：
- 空格键控制播放/暂停
- 方向键快速导航
- ESC 返回标准窗口

**适用场景**：
- 投屏到演讲厅大屏或副屏

- 透明度可调（30%-100%）

**核心显示**：

**交互特点**：

- 可拖拽移动位置

**适用场景**：
- 对着摄像头录制（题词显示在镜头下方或侧方）
- 直播主播的参考显示
- 多屏协作（一屏录制，一屏题词）

### 2.3 UI 设计规范

#### **色彩系统**

**主色板**：
- 深背景：#1E1E1E（VS Code 深灰）
- 文本主色：#E0E0E0（浅灰，高对比度）
- 强调色：#0EA5E9（天蓝色，用于按钮/高亮）
- 成功色：#22C55E（绿色，表示可用）
- 警告色：#EF4444（红色，错误/超时提示）

**应用规则**：
- 文本默认 #E0E0E0
- 次要信息 #808080（50% 灰）
- 不可用状态 #404040（深灰，降低可见度）
- 悬停高亮 +10% 亮度

#### **字体规范**

| 用途 | 字体 | 大小 | 行高 | 权重 |
|------|------|------|------|------|
| 题词显示（主） | SF Pro / 微软雅黑 | 48-72px | 1.8 | Regular |
| 题词显示（预读） | SF Pro / 微软雅黑 | 24-32px | 1.6 | Light |
| 界面标题 | SF Pro / 微软雅黑 | 16px | 1.5 | SemiBold |
| 界面内容 | SF Pro / 微软雅黑 | 13px | 1.6 | Regular |
| 说明文字 | SF Pro / 微软雅黑 | 11px | 1.5 | Light |

**特殊处理**：
- 英文和中文混合显示时，优先使用英文字体 SF Pro
- 长文本（>20字）自动调整行高至 1.8-2.0
- 移动端（平板）字体放大 20%

#### **间距规范**

- 边距标准：16px（外层）、8px（内层）
- 组件间距：12px（同级）、4px（紧密相关）
- 按钮间距：8px
- 卡片内边距：16px

#### **圆角规范**

- 主要容器：8px
- 按钮：6px
- 小组件：4px
- 悬浮框：8px

#### **阴影规范**

```
浅阴影：0 2px 8px rgba(0,0,0,0.3)
中阴影：0 4px 16px rgba(0,0,0,0.4)
深阴影：0 8px 24px rgba(0,0,0,0.5)
```

### 3.1 主工作区设计（标准窗口）

#### **3.1.1 左侧边栏 - 脚本库面板**

1. **搜索框**（48px 高）
   ```
   [🔍 搜索脚本... ]  [☰ 筛选]
   ```
   - 实时搜索（300ms 去抖）
   - 支持按标签、创建时间、使用频率筛选
   - 显示最近 5 个使用的脚本

2. **脚本列表**（可滚动区域）
   
   **单个脚本卡片**（280×80px）：
   ```
   ┌─────────────────────────────┐
   │ 📄 脚本标题 [2024.11.20]     │ ⋯
   │ 标签：销售 | 产品 | 5分钟    │
   │ ▌▌▌▌▌░░░░ 50% 完成度        │
   └─────────────────────────────┘
   ```
   
   **卡片交互**：
   - 单击：加载脚本到中央显示区
   - 右键/⋯ 菜单：编辑、复制、删除、分享
   - 拖拽：排序脚本顺序
   - 双击标题：快速重命名

3. **快捷按钮组**（48px 高）
   ```
   [+ 新建脚本] [📋 导入]
   ```

4. **分类标签栏**（可滚动，40px 高）
   ```
   [全部] [收藏] [最近] [演讲] [口播] [直播] [+ 自定义]
   ```

**交互细节**：
- 脚本未保存时，标题旁显示⚠️红点
- 正在使用的脚本用蓝色竖条标记左侧
- 列表为空时显示"没有脚本，点击新建或导入"

**显示逻辑**：

| 区域 | 显示内容 | 字号 | 透明度 | 用途 |
|------|--------|------|--------|------|
| 焦点区 | 当前发言内容 | 56px | 100% | 读词 |
| 预读1 | 下一句 | 28px | 50% | 心理准备 |
| 预读2 | 再下一句 | 20px | 30% | 整体感知 |

**智能换行算法**：
- 每行最多 15 个中文字符或 25 个英文字符
- 不在标点符号处断行（逗号、句号除外）
- 短语单位优先（同一概念不分行）
- 自动检测"。""！""？"处添加视觉停顿（+50ms 延迟显示下一行）

**焦点区域的关键指标**：
- 当前行号：顶部右侧显示"15/120"（当前行/总行数）
- 时间显示："00:45/10:00"（已用时间/总时间）
- 进度条：底部显示整体进度（100%宽度）

**编辑工具栏功能**：

| 按钮 | 功能 | 快捷键 |
|------|------|--------|
| 🔤 编辑 | 打开编辑模式，可修改脚本内容 | Ctrl+E |
| ✏️ 标记 | 标记当前位置为"容易出错"或"重点"，用红/绿色方块显示 | Ctrl+M |
| 🔍 查找 | 快速定位内容，支持关键词搜索跳转 | Ctrl+F |
| ⚙️ 设置 | 打开显示设置面板 | Ctrl+, |

## 第四部分：关键交互和用户体验细化

### 4.1 用户旅程 - 会议演讲场景

#### **阶段一：准备前（-1小时）**

**用户行为**：完成并测试发言稿

```
起点：打开应用 → 选择/新建脚本 → 加载脚本 → 测试演讲
```

**关键交互点**：

1. **脚本导入**（首次）
   - 支持格式：.txt, .docx, .pdf, .rtf
   - 导入后自动识别段落，用回车分隔
   - 弹窗提示："检测到 45 个段落，是否全部导入？"
   - 用户可预览→调整→确认

2. **脚本预处理**
   ```
   导入完成后的智能处理：
   ✓ 去除多余空格和特殊符号
   ✓ 按中文/英文自动换行
   ✓ 标记可能的停顿点（句号、问号）
   ✓ 估算总时长（基于推荐速度 100 字/分钟）
   
   显示提示：您的发言稿约 8:30，建议在 15 分钟内完成
   ```

3. **模式选择**
   ```
   系统提示："这看起来是一个 12 分钟的演讲稿。
   我们推荐使用【演讲模式】，已为您自动配置。"
   [使用推荐] [自定义模式] [了解更多]
   ```

4. **速度测试**
   - 系统建议用户朗读前 50 字，实时计算实际速度
   - 比对标准速度：
     ```
     您的实际速度：92 字/分钟
     ```

### 5.2 表现力评分算法

**综合评分公式**：

```
总体评分 = 停顿流畅度(30%) + 节奏稳定性(30%) + 连贯性(25%) + 速度控制(15%)

停顿流畅度 = 1 - (非预期停顿次数 / 总段落数) × 系数
节奏稳定性 = 1 - (实际速度波动 / 推荐速度范围) × 系数
连贯性 = 1 - (返工次数 / 总段落数) × 系数
速度控制 = 1 - (|实际平均速度 - 推荐速度| / 推荐速度)

所有指标最终映射到 1-10 分
```

**评分等级**：
- 9.0-10.0 分：专业级（可用于重要场合）
- 8.0-8.9 分：优秀（适合一般演讲）
- 7.0-7.9 分：良好（需要练习改进）
- 6.0-6.9 分：合格（继续练习）
- <6.0 分：需要改进（建议重新练习）

## 附录 C：UI 组件库规范

### C.1 按钮设计

**按钮类型**：

1. **主要按钮**（Primary）
   - 背景色：#0EA5E9（天蓝色）
   - 悬停：#0284C7（深蓝）
   - 禁用：#808080（灰色，50% 透明）
   - 圆角：6px
   - 内边距：12px 24px

2. **次要按钮**（Secondary）
   - 背景色：transparent
   - 边框色：#808080
   - 文字色：#E0E0E0
   - 悬停：背景色 #404040

3. **危险按钮**（Danger）
   - 背景色：#EF4444（红色）
   - 悬停：#DC2626（深红）
   - 用于删除操作
